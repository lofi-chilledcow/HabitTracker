Request → Handler Mapping

Every HTTP request resolves to exactly one MediatR message and one handler

Method Endpoint Type MediatR Message Handler DB Operation
GET /api/habits Query GetAllHabitsQuery GetAllHabitsQueryHandler SELECT * (AsNoTracking)
GET /api/habits/{id} Query GetHabitByIdQuery GetHabitByIdQueryHandler SELECT WHERE Id (AsNoTracking)
POST /api/habits Command CreateHabitCommand CreateHabitCommandHandler INSERT + SaveChanges
PUT /api/habits/{id} Command UpdateHabitCommand UpdateHabitCommandHandler UPDATE + SaveChanges
DELETE /api/habits/{id} Command DeleteHabitCommand DeleteHabitCommandHandler DELETE + SaveChanges
Legend
HTTP layer — thin controller, delegates only
MediatR bus — decouples caller from handler
Command — mutates state, uses change tracking
Query — read-only, AsNoTracking for performance
Command handler — owns transaction via SaveChangesAsync
Query handler — projects directly to DTO, no entity hydration overhead
HabitDbContext — single EF Core context, scoped lifetime
SQL Server — dbo.Habits table, schema managed by EF migrations
